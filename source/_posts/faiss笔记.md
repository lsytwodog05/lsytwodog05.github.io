# FAISS 向量检索库笔记

## 一、FAISS 是什么？

FAISS（Facebook AI Similarity Search）是一个专门用于**在海量高维向量中快速寻找相似项**的工具库。

它由 Facebook AI Research 开源，核心目标：**快 + 能扛大数据**。

### 什么是“数字向量”？

- 一串浮点数，例如 `[0.23, -1.5, 0.88, …]`
- 现代 AI（大模型、图像识别、推荐系统）会将文字、图片、音频等转化为向量。
- 两个东西越相似，它们的向量在空间里就越“靠近”。

---

## 二、FAISS 解决什么问题？

**场景**：你有 100 万个向量，现在给你一个新的向量，问“哪 10 个最像它？”

- 普通方法（逐条计算距离）可能需要几秒甚至几分钟。
- FAISS 能在 **几毫秒到几十毫秒** 内给出结果，且精度损失极小。

**核心价值**：**大规模向量检索的高效实现**。

---

## 三、主要功能

1. **快速搜索 Top-K 相似向量**
   - 输入一个向量 → 返回最相似的 K 个（例如前 5 个）
   - 支持两种距离/相似度标准：
     - **L2 距离（欧氏距离）**：数值越接近越相似（默认）
     - **内积（IP）**：方向越一致越相似（常用于语义搜索，需提前归一化）

2. **支持超大数据集**
   - 可处理百万、千万甚至上亿的向量
   - 内存不够时，可使用**量化压缩**（如 PQ）来减小内存占用（牺牲少量精度）

3. **GPU 加速**
   - 使用 `faiss-gpu` 版本，在 NVIDIA 显卡上可再提速 10 倍以上

4. **给向量绑定自定义 ID**
   - 默认返回的是向量在索引中的位置（序号）
   - 你可以传入自定义 ID（如数据库主键、商品 SKU），搜索时直接返回业务 ID

5. **持久化（存盘 & 加载）**
   - 索引可保存到硬盘，下次启动直接加载，无需重建

---

## 四、FAISS 不能做什么？（常见误区）

- **不能动态频繁增删向量**
  - 大部分高效索引（如 IVF、HNSW）建好后不支持随意插入或删除。
  - 应对策略：
    - 定期重建索引（适合每天/每周更新）
    - 使用简单索引（如 IndexFlat，但慢）
    - 自己分层管理（新数据放内存，老数据放 FAISS）

- **不是一个数据库**
  - FAISS 只负责向量相似度计算，**不支持业务条件过滤**（如“价格<100的商品”）。
  - **实践做法**：FAISS 召回候选集 → 数据库二次过滤 + 排序。

## 五、安装方法

| 情况 | 命令 |
|------|------|
| 只有 CPU（大多数电脑） | `pip install faiss-cpu` |
| 有 NVIDIA 显卡 + CUDA | `pip install faiss-gpu` |

> 不要同时安装两个版本，会冲突。不确定时先用 `faiss-cpu`。

## 六、完整代码示例（使用 IndexFlatIP + 余弦相似度）

```python
# 第 1 步：导入库
import faiss
import numpy as np

# 第 2 步：准备已有数据（10,000 个 768 维向量，模拟 BERT 句子向量）
database_vectors = np.random.rand(10_000, 768).astype('float32')

# 第 3 步：创建索引（IndexFlatIP 表示内积，适合语义相似度）
index = faiss.IndexFlatIP(768)   # 768 是向量维度

# 第 4 步：【关键】使用内积必须预先对所有向量做 L2 归一化
faiss.normalize_L2(database_vectors)

# 第 5 步：将向量加入索引（建库）
index.add(database_vectors)
print(f"已加载 {index.ntotal} 个向量")

# 第 6 步：准备查询向量（也需归一化）
query_vector = np.random.rand(1, 768).astype('float32')
faiss.normalize_L2(query_vector)

# 第 7 步：搜索 Top-3
k = 3
similarities, indices = index.search(query_vector, k)

# 第 8 步：查看结果
print("最相似的 3 个向量在数据库中的位置：", indices[0])
print("对应的相似度分数（越接近 1 越相似）：", similarities[0])
```

### 输出示例

```
已加载 10000 个向量
最相似的 3 个向量在数据库中的位置： [9284 1897 9108]
对应的相似度分数： [0.7853632 0.78344375 0.7832435]
```

## 七、如何选择合适的索引？

| 数据量级 | 关注点 | 推荐索引 | 说明 |
|----------|--------|----------|------|
| < 1 万条 | 绝对准确（实验/小工具） | `IndexFlatL2` 或 `IndexFlatIP` | 暴力搜索，结果 100% 准确 |
| 1 万 ~ 100 万条 | 速度快，允许轻微误差 | `IndexIVFFlat` | 先聚类分组，搜索时只查相关几组，速度提升 10~100 倍 |
| > 100 万条 | 节省内存 + 响应快 | `IndexIVFPQ` 或 `IndexHNSW` | IVFPQ 量化压缩；HNSW 用图结构加速，两者都牺牲少量精度 |
| 有 NVIDIA GPU | 榨干性能 | 安装 `faiss-gpu`，使用 IVF 系列 | 获得额外加速 |

---

## 八、L2 距离 vs 内积（IP）如何选择？

- **IndexFlatL2**：向量**未归一化**，或关心“数值差距”（如图像像素特征）时使用。
- **IndexFlatIP**：向量**已经 L2 归一化**，且关心“方向/语义相似度”（如文本 embedding）时使用。

> 归一化后，内积 = 余弦相似度，范围 [-1, 1]。

---

## 九、常见索引类型速查

| 索引类型 | 工作方式 | 适用场景 |
|----------|----------|----------|
| **IndexFlatL2** | 暴力计算欧氏距离 | 小数据集，要求精确 |
| **IndexFlatIP** | 暴力计算内积（需归一化） | 小数据集，语义相似度 |
| **IndexIVFFlat** | 先聚类（分组），组内暴力 | 中等规模，需加速 |
| **IndexIVFPQ** | 聚类 + 向量量化压缩 | 超大规模，内存有限 |
| **IndexHNSW** | 基于图结构的近似搜索 | 超大规模，要求极快响应 |

> 带 `IVF` 的索引可以通过参数 `nprobe` 控制搜索时探查的聚类数，越大越准但越慢。

## 十、在 LangChain 中的使用（简化封装）

LangChain 为方便用户，隐藏了 FAISS 的细节：

- 使用 `FAISS.from_texts()` 创建索引时，默认自动选择 `IndexFlatL2`。
- **优点**：简单、精确，适合小到中型数据集（几万~几十万条）。
- **缺点**：数据量达百万级别时，搜索速度会明显下降。

---

## 十一、总结

- **FAISS = 向量搜索引擎**
- **核心能力**：快 + 准 + 能扛大数据
- **典型应用**：RAG 知识召回、推荐系统、以图搜图、文本匹配等

> 实际工程中，经常采用“**FAISS 召回候选 + 数据库二次过滤**”的组合架构。
